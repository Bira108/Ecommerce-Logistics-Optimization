## 📋 Project Overview
This project identifies geographic and product-level **"profit leakage"** caused by outdated flat-rate shipping models. By analyzing a pilot dataset of e-commerce transactions, I engineered a **Volume-Based Tiered Shipping Model** to recover margins and incentivize bulk purchasing.

## ⚙️ The Process
<ul>
  <li><b>Data Engineering:</b> Standardized inconsistent geographic data (e.g., "IN." to "Indiana") and joined fragmented sales/shipping tables using SQL.</li>
  <li><b>Audit & Validation:</b> Performed a Cross-Table Audit to ensure 100% relational integrity across regions.</li>
  <li><b>Scenario Modeling:</b> Created calculated fields in Tableau to compare the <b>Current State</b> against a <b>Hypothesis Model</b> (using quantity-based multipliers from 1.0 down to 0.3).</li>
  <li><b>UI/UX Design:</b> Built a stakeholder-ready dashboard with global filters for Category and State exploration.</li>
</ul>

---

## 💡 Key Insights
* <b>Identified Leakage:</b> Discovered that heavy categories (Pet Food/Electronics) in long-distance states like <b>Washington</b> were creating a <b>65% shipping-to-revenue ratio</b>.
* <b>Product Impact:</b> Isolated a single high-volume product causing a <b>$5,000 annual loss</b> due to unoptimized shipping.
* <b>Profit Recovery:</b> Demonstrated that a tiered shipping strategy increases total net profit from <b>$250K to $411K</b>—a <span style="color:green"><b>64% increase</b></span>—without changing product prices.

---

## 🚀 Recommendation
Based on the data, I recommend a transition to the **Hypothesis Shipping Model**. This turns logistics from a cost-center into a competitive advantage by aligning shipping fees with real-world carrier efficiencies.
